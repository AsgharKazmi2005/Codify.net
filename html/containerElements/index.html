Container Elements

Container Elements are a concept that I wish I had learned earlier in my career, but they were not taught to me in a sensible way, like I will teach you. These elements belong together in their own unit because they all do a similar job, so instead of scattering them around, here are all of them at once. Container Elements do not show any content themselves. Their job is to act as a container for elements that do show content like headers, images and paragraphs. They are critical to the html language because without them our webpages would be messy, frustrating and inconsistant. The format for container elements are the same as normal elements and container element are NEVER self closing. As a good rule of thumb, every HTML element should be inside of a container element, so it is important to know what container elements are and how to use them. Let's get into learning about them.

Divs

Divs are the most popular container elements. Short for divisions, they are the general multi-purpose container element that you will be using most of the time. If you are not sure which of the container elements to use, chances are a div will do the job, that is how multi-purpose they are. Headers & Footers

Similar to headers and footers on a word document, headers and footers in html are the containers we will use for the top and bottom of our webpage. It is good practice to start your website with a header element and end it with a footer element. The syntax's are as follows: <header></header> and <footer></footer>. Functionally, they work the same as a div, but they help us know which div is at the top and which div is at the bottom. This is called a semantic element, meaning that the name of the element describes what it is. A div is not a semantic element because as a reader, we have no clue what the div contains. However, if we see a header or footer element, we can made a good inference as to what they contain just from the name alone. You should always use a semantic element over a non-semantic element whenever you can because it makes your code easier to read. Let's add headers and footers to our code:

Again, using CSS we will make this web page look much better, but for now, this basic structure is fine. As you can see, the header element is used for putting the website name and navigation. Headers commonly include login links, registration links and search bars among other links in their headers. Footers are used for content that isnt essential for user experience like policies and documentation. Footers commonly include Terms of Service notices, Privacy Policy notices, Cookies notices, DMCA (Copyright) notices, social media links, links to use the website in other languages, about us pages and other pages that should not be at the top of the page. All in all, headers and footers are incredibly useful and we must use them on your web pages.

Sections, Articles & Aside

Sections, Articles and Asides all fall into a similar boat when it comes to semantic elements. They all define parts of the site that are not the main focus, but nonetheless part of the page. Sections are more general and can be placed anywhere on the page so long as they are not the central focus. Articles are much like sections and they should not be the main part of a page. Asides are elements that are strictly on either side of the page. Of course you could use these elements in places they should not be, but this is a terrible practice and it will confuse people who read your code. Spans

Spans are another commonly used element. Usually, spans are used for containing text. They are much like divs, infact the only difference is that divs take up the whole line whereas spans only take-up as much space as they need. In technical terms, a div is a block element (meaning it takes up the whole line) and spans are inline elements (they take up as much space as they need and no more). Be aware that we can switch between inline and block using css. Here is an example of using the <span></span> element:
