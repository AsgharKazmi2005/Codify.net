Basic Elements

On this page you will find all the commonly used basic HTML elements. HTML elements are tags that we can use to display something on a page. There are dozens of elements in HTML and we will learn most of them. Let's start where everyone starts, Headers!

HTML Headers

HTML Headers are exactly what they sound like, heading. There are six headers in HTML and they all are different sizes. The largest is the <h1></h1> header. Next would be the <h2></h2>, then the <h3></h3>, so on and so forth until the final header element, <h6></h6>. These header elements make the text bold as well. To make any text show on our webpage, we can use these elements. For example <h4>Hello World</h4> will print 'Hello World' on our webpage. Try to write a sentence with every type of header element. HTML Paragraphs

The next element we will learn about is the paragraph element. Infact, this paragraph you are reading right now is a paragraph element. A paragraph element looks like this: <p></p> but with text in between the tags. It is used to write paragraphs such as the one you are reading right now or the ones they have in news stories. Let's add a paragraph element below the header elements and see how they compare. As you can see, a paragraph in basic form is around the size of an h4 element and it is not bolded at all. Ofcourse we can change the size of our paragraph element but we will learn to do that when we get to CSS, for now this is perfect.

HTML Hyperlinks

Have you ever seen blue text that links from one site to another, that's a hyperlink! In HTML, we call a hyperlink an anchor element. But there is one difference that an anchor tag has that both the paragraph and headers do not. It requires an attribute. We will learn more about attributes in a later section, but all you need to know now is that they help us use html elements. The syntax for an anchor element is as follows: <a href="www.google.com">Visit Google!</a>. The attribute we are using here is the href attribute. href stands for Hypertext Reference. What this means is that it tells the anchor element which website it should link to. It is important to note that you can put anchor tags inside of paragraph and heading tags like so: <p>Make Sure To <a href="www.google.com">Visit Google!</a></p>. In this example, the 'Visit Google!' text will link to the google website, and the rest of the text will be plain text. HTML Images

Images are a very popular element as they can be used to show pictures, much like you have seen in these pages. The syntax for an image tag is as follows: <img src="linkToImage.png">. That's right, the img tag is self closing and requires a src attribute. The src attribute stands for source and it tells the img element where the image is actually located so it can find and display it. There are two ways to link an image, Locally, and From the internet (non-local). Let's talk about non-local first as that is going to be compatible with repl and VSCode. What you need to do is go to google and search up any image of your choice, I will be looking for a Unicorn image to match my earlier paragraph. Once you have gone through google images (or any other browser), you want to right click on the image and click 'copy image address'. This 'address' works much like the address of your home. Your address tells people where to find you, in this case, the image address tells the img element where to find the image. Add an image to your project, it should look like so:

For now, let's look at the second method. To locally source images, you need a device that can download files, if your using Repl, you cannot do this method. Okay, First we need to create a folder, name it anything you want. Next, copy your index.html file into the new folder. Then close VSCode and open it again. This time, select the 'Open Folder' Option and choose the folder we just made. Finally, download a picture of your choice from the internet and drag it into that new folder. Now all you have to do is put the name of the file in between the src="" attribute and you have locally sourced the img element. This method is preferred among web developers because the links are much shorter and you don't have to deal with extremely long links like you do with the other method. HTML Comments

There is one more critical thing you must know about HTML, comments. Now comments are not an element at all, they are a way of writing inside HTML files without executing the writing. Essentially, HTML comments are like writing notes, they help professional web developers stay organized by adding comments above elements. Also, if you want to stop an element from showing on the webpage, but do not want to delete it because you plan to use it later, you can comment it out. To comment in html, we use <!-- --> and put text in between. On most html editors, the shortcut CTRL + / will add a comment. Selecting html and then pressing CTRL + / will comment the selected portion out. Here are Some examples in play:

Comments are seriously important when you are working in a group setting. Most computer science jobs require you to work with a team and if you are reading one of your colleague's code, you are expecting to see comments along with the code so you know what each part of the code does.
