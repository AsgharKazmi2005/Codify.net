HTML Forms

HTML Forms are an integral part of modern webpages. Whether it is simply submitting a search or signing in to a site, form are completely necessary. The difficult part is that HTML forms come with many elements and attributes so it can be difficult to learn. On this page you will find most of the elements along with the optional attribute parameters. This is a difficult section so we will take it one element at a time. Let's begin with the form container element.

Form Element

As we talked about briefly in the previous section, Form elements are the containers that we are going to use for all our forms. To begin we are simply going to create the form element, at this point nothing should show on your screen as we have not added any visual elements. When we create forms, we have to assign an action attibute. Essentially what it does is once the form is complete and the user hits submit, the data is sent to the specified action URL where it is processed. We will learn to do this way later on in the node.js course. For now we will not specify an action attribute to our form as we are simply learning the elements.

Form Attributes

That was the final form element. Now before we end the unit off, let's go over some old attributes and learn some new ones that will make our forms more reliable. Right now our users can set their ages to negative numbers and can submit the form while leaving everything blank. Luckily, HTML has many built in attributes so we can get the results we want without our users sending incorrect information to our servers. Let's begin.

Name Attribute

The 'name' attribute as we have used many times in this unit simply gives a name value to an element. It is good practice to apply the name attribute to EVERYTHING you can. It helps you understand your own code and helps others understand your code. It is compatible with all form elements.

Id Attribute

Next is the 'ID' Attribute which we have also used many times. We use IDs to link labels and inputs together as well as to link datalists and inputs together. It is recommended you apply the Id attribute to all your form elements as they help us keep our code clean as well as make it easier to use CSS and Javascript in the future. It is compatible with all form elements.

For & List Attributes

The 'for' attribute allows us to link a label to another element. It should really only be used with the label element. The list attribute allows us to link an input to a datalist. It should only be used on the input element.

Once the user click on the text box, the placeholder text dissapears and the user can write their own text. Required Attribute

Earlier we had the issue that people could submit the webpage without filling out all the information. By adding the 'required' attribute to our elements, we can make it so the user cannot submit the form until all the required information is filled out. The 'required' attribute is a boolean attribute meaning that is is true or false. Any element is either required or not required. By default every element is not required so if we want to require it, all we have to do is put the word 'required' inside of our element tags. No ="" is necessary here. If a value is not required, we do not have to do anything to our element. It is compatible with text, search, url, tel, email, password, number, checkbox, radio, file and all time input types. It also works with textareas, datalists and select elements. Let's add the required attribute to all our elements since all the information is required.

Value Attribute

The 'value' attribute works similar to the placeholder attribute but the content acts as if the user has input it. When we use a placeholder, the text inside dissapears. By setting the value attribute, instead of dissapearing, the text is actually there. Usually the user has to erase the text to put their own text in so It is not reccomended. For the "reset" and "submit" input types it defines the text on the button. For text inputs it defines the initial value of the input field. For "checkbox" and "radio" it defines the value associated with the input (setting the value on our checkbox or radio buttons will send that value to our server so every checkbox and radio should have a value).

As you can see, if we try to submit a blank value, HTML has it's own error message. Of course we can make our own custom error message but the default HTML one does the job. Now that solves the problem of users submitting blank inputs.

The multiple attribute is a simple boolean that allows you to submit multiple different values for a single input. It is only compatible with the email and file input types. It is not a very commonly used attribute.

The pattern attribute as we discussed early allows us to set how the user input should be format. We must write either [0-9] (This stand for any number) or [A-Za-z] (Which stands for any alphabet) followed by {} with a number inside denoting how long the rule should be. For example, if pattern="[A-Za-z]{5}[0-9]{3}" then we must enter 5 letters followed by 3 numbers. Additionally if we want to restrict the user to upper case we can use only [A-Z] and for lowercase, only [a-z].The pattern attribute is compatible with the text, tel, email, url, password, and search input types. The step attribute is an attribute that allows us to set an increment to a number, date or range input type. For example, let's say we want our users to enter the approximate amount of months that have passed since their last appointment and want to make it so the values they submit are multiples of three (ie. 3 months, 6 months, 9 months, 12 months, etc.), we can set step="3" and our users will be forced to set their answer to a multiple of three. It is used with the date, month, week, time, datetime-local, number and range types. It is uncommon with time input types though. Autofocus Attribute

The autofocus attribute decides which element is active on page load. Usually, you attach the autofocus attribute on the first input on the form so when the user loads the form, they can type into the first input element without having to click it. It works on all inputs, but in most cases, it is used with a textbox. Add the autofocus attribute to the first input on your form.

It is critical that you only have one autofocus attribute per form. If you have more than one, then the element that is earliest in the HTML file will have autofocus and the rest will not. Only use autofocus on the first element.
